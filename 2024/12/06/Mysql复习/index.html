3. 建表

约束类型 作用 示例
NOT NULL 确保列值不能为空 name VARCHAR(50) NOT NULL
UNIQUE 确保列值唯一 email VARCHAR(100) UNIQUE
PRIMARY KEY 唯一标识一条记录，不能为空 id INT PRIMARY KEY
FOREIGN KEY 定义表之间的关系，引用另一表的主键 FOREIGN KEY (user_id) REFERENCES users(id)
DEFAULT 为列指定默认值 is_active BOOLEAN DEFAULT 1
CHECK 定义列值的合法范围或条件 age INT CHECK (age >= 18)
索引类型 特点 限制 适用场景
普通索引 - 不强制唯一性
- 用于加速查询		 - 可重复，不能防止重复数据 用于频繁查询但无唯一性要求的列，例如分类字段
唯一索引 - 保证列值唯一
- 用于加速查询		 - 列值必须唯一，允许空值 确保列值唯一，例如用户名、邮箱等唯一标识
主键索引 - 特殊的唯一索引
- 列值不能为空
- 默认创建聚集索引		 - 每个表只能有一个主键 唯一标识每行数据，例如学号、身份证号等
聚集索引 - 数据行存储顺序与索引顺序一致
- 每个表只能有一个聚集索引		 - 必须与数据存储关联，影响插入性能 用于加速基于主键或特定字段排序的频繁查询

当用主键约束和唯一约束时创建表或者修添表约束，会自动的添加索引，删除同理。外键不会，但关联的那个键必须要有索引。

如果你需要备注信息comment即可，注释字段：定义好字段约束后面写，注释表：全部定义好最后写

4. 插入

Affected rows 是反馈信息，表示 插入、更新或删除操作影响的行数

不需要合并外键写法

父表是被引用的表，通常包含主键或者唯一键，其他表通过外键来引用它。

子表是引用父表的表，包含外键列，指向父表的主键或唯一键。

5. 删除模式

RESTRICT（默认）：如果父表中的数据有子表依赖，删除父表数据时会阻止删除操作。

**CASCADE**：如果父表中的数据有子表依赖，删除父表数据时会自动删除子表中的相关数据。

MySQL 不允许直接修改外键约束。你必须先删除现有的外键约束，然后重新添加一个新的外键约束，指定新的删除规则。
